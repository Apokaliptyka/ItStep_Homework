# Проект "Управління студентами"

## Опис проекту

Проект "Управління студентами" розроблено для систематизації та зберігання інформації про студентів та їх академічну успішність в рамках навчального закладу. Система дозволяє додавати, видаляти та редагувати інформацію про студентів та курси, а також виконувати операції з оцінками.

## Патерни та принципи SOLID

У проекті використано принципи SOLID:
- **Принцип одиничної відповідальності (SRP)**: Кожен клас відповідає за одну конкретну функціональність.
- **Принцип відкритості/закритості (OCP)**: Класи проекту розроблені таким чином, щоб вони були відкриті для розширення, але закриті для модифікації.
- **Принцип заміщення Лісков (LSP)**: Усі класи підкласів можуть замінювати базовий клас без змін у коректності програми.
- **Принцип інверсії залежностей (DIP)**: Залежності в проекті орієнтовані на абстракції, а не на конкретні реалізації.

## Документація за класами

- **Output (Виведення)**: Абстрактний клас, що визначає метод display, призначений для виведення інформації про студента.
- **ConsoleOutput (Виведення у консоль)**: Реалізація класу Output, яка виводить інформацію про студента у консоль.
- **FileOutput (Виведення у файл)**: Реалізація класу Output, яка записує інформацію про студента у файл.
- **Student (Студент)**: Клас, що представляє студента. Зберігає основну інформацію про студента та інформацію про курси та оцінки.
- **Course (Курс)**: Клас, що представляє курс. Зберігає назву курсу та список студентів, які записані на цей курс.
- **Grades (Оцінки)**: Клас, що представляє оцінки студента. Забезпечує можливість додавання та оновлення оцінок.
- **School (Школа)**: Клас, який управляє студентами та курсами. Надає можливість реєстрації студентів на курси, видалення та додавання студентів та курсів, а також операції з оцінками.

## Патерн "Стратегія"

У проекті використано патерн Стратегія (Strategy). Цей патерн дозволяє вибирати алгоритм виконання певної задачі на льоту. Наприклад, для виведення інформації про студента можна використовувати різні стратегії виведення, такі як виведення у консоль або у файл. Клас Student використовує абстракцію Output, дозволяючи легко змінювати спосіб виведення інформації про студента без зміни самого класу.
